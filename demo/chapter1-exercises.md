# 第一章习题答案

## 1. Case 分析

### Case A
不是智能体。它具备极强算力，但本身不直接感知环境、制定目标并自主行动，更像计算基础设施而不是 agent。

### Case B
是智能体。
- 按传统分类，属于基于模型的反应式智能体，兼具一定目标导向能力。
- 按应用形态，属于具身智能体。
- 原因是它能持续感知路况、在毫秒级做出制动或变道动作，并以安全驾驶为目标闭环运行。

### Case C
是智能体。
- 属于目标驱动 / 效用驱动型智能体。
- 也可以看作规划型智能体。
- 它会基于棋局状态评估未来多步收益并选择最优动作。

### Case D
是智能体。
- 属于大语言模型驱动的工具型智能体。
- 同时带有工作流执行与规划决策特征。
- 它需要理解语言、查询外部系统、分析原因并输出安抚与解决方案。

## 2. 智能健身教练的 PEAS

### Performance measure
- 用户训练目标达成率
- 动作标准度
- 心率和负荷是否处于安全区间
- 用户坚持率、满意度
- 饮食建议执行效果

### Environment
- 用户身体状态
- 家庭 / 健身房训练场景
- 可穿戴设备
- 饮食与作息环境
- 外部噪音、场地限制、器械条件

### Actuators
- 语音提示
- 屏幕文字 / 图表反馈
- 动态调整训练计划
- 推送饮食建议
- 发出风险预警

### Sensors
- 心率、步频、卡路里、血氧等可穿戴数据
- 摄像头姿态识别
- 用户主观反馈
- 历史训练记录
- 体重、体脂、围度等周期数据

### 环境特征分析
- 部分可观测：无法完整掌握用户全部生理与心理状态
- 随机性：身体反应、疲劳、睡眠等都带来不确定性
- 动态性：训练过程中身体状态实时变化
- 序列性：今天的训练会影响后续恢复和计划
- 多智能体：用户、教练、设备平台都在影响系统

## 3. Workflow vs Agent for 退款

### Workflow 的优点
- 规则清晰，可审计，可预测
- 成本低，适合高频标准场景
- 风险容易控制

### Workflow 的缺点
- 对复杂例外情况不灵活
- 规则维护成本会随业务膨胀
- 难以利用上下文做细致判断

### Agent 的优点
- 能综合历史行为、商品状态、上下文信息
- 对复杂长尾问题更灵活
- 能持续优化决策质量和交互体验

### Agent 的缺点
- 可解释性和稳定性较弱
- 合规和误判风险更高
- 评估与上线成本更高

### 适用场景
- Workflow 更适合规则稳定、金额明确、合规要求高的退款场景。
- Agent 更适合高复杂度、需要上下文判断、客服成本高的复杂售后场景。

### 我的选择
我会优先选 Workflow 作为主干，因为退款属于高风险决策场景；再在复杂 case 上引入 Agent 做辅助判断。

### Approach C
采用混合方案：
- 第一层由 Workflow 做硬规则过滤与自动审批
- 第二层由 Agent 处理边缘案例、材料总结、风险评分与客服建议
- 最终高风险 case 交给人工审核

这样可以兼顾稳定性、效率与灵活性。

## 4. 旅行助手扩展思路

### 记忆能力
- 增加 `memory` 结构保存用户偏好、预算、历史接受 / 拒绝记录。
- 每轮构造 prompt 时把记忆摘要拼进去。
- 新 observation 中如果出现新的偏好信号，就更新 memory。

### 售罄自动推荐替代项
- 新增 `check_ticket_availability` 工具。
- 推荐景点后先检查余票；若售罄，则进入新一轮 Thought，调用候补推荐工具或重新搜索。

### 连续 3 次拒绝后调整策略
- 记录 `rejection_count`。
- 当用户连续拒绝达到 3 次时，将 observation 写入“当前策略无效，需要切换维度”，例如从热门景点转向小众 / 室内 / 低预算方案。

## 5. System 1 / System 2 示例：医疗诊断助手

### System 1 负责
- 识别症状模式
- 从影像或检验指标中快速发现异常信号
- 对常见病例做初筛

### System 2 负责
- 结合病史、药物史、家族史做因果推理
- 排除冲突诊断
- 生成检查建议与治疗方案说明

### 协同方式
- 先由 System 1 快速召回高概率候选诊断
- 再由 System 2 对候选结果进行逻辑审查、证据整合和风险提示
- 最终输出更稳健的诊断建议

## 6. Agent 局限性分析

### 为什么会幻觉
- 语言模型本质是概率生成，不天然等价于事实检索
- 训练数据不完整或过时
- 工具返回信息不足，模型会补全“看起来合理”的内容
- prompt 约束不够强时，模型更容易自由发挥

### 不限制循环次数会怎样
- 可能陷入无限循环
- 重复调用工具，增加成本和时延
- 在错误 observation 上不断放大偏差
- 用户体验变差，系统稳定性下降

### 如何评估智能体智能水平
不能只看准确率，还应综合：
- 任务完成率
- 工具调用成功率
- 平均步数 / 成本 / 延迟
- 面对异常和不确定性的恢复能力
- 结果可解释性
- 用户满意度
- 长任务中的稳定性与一致性
